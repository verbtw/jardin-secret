# Jardin Secret: аккаунты и карточки ароматов

## Цель

Добавить настоящую регистрацию покупателей и превратить карточки всех 114 товаров в полноценные страницы ароматов. Оплата остаётся вне сайта: оформленный заказ по-прежнему передаётся менеджеру в Telegram.

## 1. Регистрация и профиль

### Архитектура

- Supabase Auth отвечает за регистрацию по email и паролю, вход, выход, подтверждение email и восстановление пароля.
- Таблица `profiles` хранит имя, телефон, город и адрес доставки. Строка профиля связана с `auth.users.id`.
- Row Level Security разрешает пользователю читать, создавать и изменять только собственный профиль.
- React-контекст `AuthProvider` хранит сессию и предоставляет единый API страницам.
- Публичный каталог доступен без аккаунта. Страница `/account` защищена и отправляет гостя на `/login`.

### Интерфейс

- В шапке появляется компактная кнопка аккаунта.
- `/register` — email, пароль, повтор пароля и согласие с обработкой данных.
- `/login` — вход и ссылка на восстановление пароля.
- `/forgot-password` и `/reset-password` — запрос письма и установка нового пароля.
- `/account` — редактирование имени, телефона, города и адреса, выход из аккаунта.
- После входа пользователь возвращается на страницу, с которой начал авторизацию.
- Checkout подставляет данные профиля. Пользователь может изменить их только для текущего заказа либо сохранить обратно в профиль явным действием.

### Состояния и ошибки

- Формы показывают загрузку и блокируют повторную отправку.
- Ошибки Supabase преобразуются в короткие русские сообщения без технических деталей.
- После регистрации показывается просьба подтвердить email.
- При недоступном Supabase каталог, корзина и заказ через Telegram продолжают работать; недоступен только аккаунт.
- Секретный service-role ключ никогда не попадает во frontend. Используются только публичные `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`.

## 2. Полные данные ароматов

### Модель данных

Каждый товар получает структурированные поля:

- `editorialDescription` — уникальное нейтральное описание своими словами;
- `fragranceFamily` — парфюмерное семейство;
- `topNotes`, `heartNotes`, `baseNotes` — пирамида нот;
- `keyAccords` — основные аккорды;
- `perfumers` — парфюмеры, если подтверждены;
- `launchYear` — год выпуска, если подтверждён;
- `detailsSourceUrl` — источник характеристик;
- `detailsStatus` — `verified` или `partial`.

Исходное описание и ссылка на публикацию Telegram сохраняются отдельно как коммерческая информация магазина.

### Источники и точность

- Первый приоритет — официальная страница бренда.
- Если официальная страница недоступна или удалена, допускается проверенный парфюмерный каталог.
- Тексты источников не копируются: описание формулируется заново и кратко.
- Неподтверждённые год, парфюмер или отдельный уровень пирамиды не выдумываются и не показываются.
- Для каждого из 114 товаров обязательно есть описание, семейство и хотя бы список ключевых нот. Полная пирамида показывается там, где она подтверждена.

### Карточка товара

- Сначала остаются цена, объём, наличие и кнопки заказа.
- Затем идёт редакционное описание аромата.
- Ноты визуально разделены на верхние, сердце и базу; при частичных данных показывается один блок «Ключевые ноты».
- Отдельно показываются семейство, аккорды, год и парфюмер, если данные есть.
- Ссылка «Источник характеристик» открывает страницу источника, а ссылка на Telegram остаётся ссылкой на предложение магазина.

## 3. Данные и обновление

- Структурированные характеристики хранятся рядом с каталогом и объединяются с импортированными Telegram-товарами по стабильному `id`.
- Повторный импорт Telegram не затирает редакционные данные.
- Скрипт проверки отклоняет неизвестный `productId`, пустое обязательное описание и товар без семейства или нот.
- Если один аромат повторяется в нескольких публикациях, характеристики переиспользуются, но цена и наличие остаются привязанными к конкретной карточке.

## 4. Проверка

- Unit-тесты: авторизационные ошибки, профиль, объединение данных и валидация всех 114 ароматов.
- Component-тесты: регистрация, вход, профиль, автозаполнение checkout и отображение полной/частичной пирамиды нот.
- E2E: регистрация и вход в тестовой среде, редактирование профиля, заполненный checkout, открытие карточки аромата.
- Production-проверка: главная, каталог, прямой URL товара, изображения, аккаунт и отсутствие ошибок консоли на desktop/mobile.

## 5. Публикация

- SQL-миграция и переменные окружения настраиваются в Supabase/Vercel.
- Изменения публикуются в существующий проект Vercel.
- Готовая ветка отправляется в публичный GitHub-репозиторий `verbtw/jardin-secret`.
- Локальная папка `portfolio-images/` остаётся вне Git и деплоя.
