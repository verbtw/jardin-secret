# Jardin Secret: отзывы из Telegram и с сайта

## Цель

Показать на сайте текстовые отзывы из канала `@jardinotzivi` и разрешить авторизованным покупателям оставлять собственный отзыв после выполненного заказа. Фото и видео не импортируются и не хранятся.

## Источники отзывов

### Telegram

- Отдельный импортёр читает публичную web-версию `https://t.me/s/jardinotzivi`.
- Сохраняются только текст публикации, дата, ID и ссылка на исходный пост.
- Медиа, подписи файлов и технические сообщения без содержательного текста отбрасываются.
- Текст очищается от служебных призывов и повторяющихся ссылок, но смысл и формулировки клиента не переписываются.
- Если имя автора нельзя уверенно определить из текста, показывается нейтральное имя «Покупатель Jardin Secret».
- Повторный импорт обновляет данные по `sourcePostId` и не создаёт дубликаты.

### Отзывы с сайта

- Оставить отзыв может только пользователь, вошедший в аккаунт.
- При оформлении заказа авторизованным пользователем создаётся строка `orders` со снимком состава корзины и публичным кодом заказа. Telegram остаётся основным каналом общения и оплаты.
- Новый заказ имеет статус `pending`. Менеджер после выполнения меняет статус на `completed` через Supabase Dashboard; отдельная административная панель не входит в первый релиз.
- Форма отзыва доступна только для `completed` заказа этого пользователя, по одному отзыву на заказ.
- Отзыв содержит оценку 1–5, текст от 20 до 1500 символов и необязательную привязку к одному аромату из заказа.
- Новый отзыв получает статус `pending` и становится публичным только после модерации менеджером через Supabase Dashboard.

## Данные и безопасность

- Таблица `orders`: владелец, публичный код, статус, снимок товаров, дата создания и выполнения.
- Таблица `reviews`: владелец, заказ, необязательный товар, оценка, текст, статус и даты.
- RLS разрешает покупателю читать собственные заказы, создавать один отзыв только к собственному выполненному заказу и читать собственные отзывы.
- Публично доступны только отзывы со статусом `published`; email, телефон, адрес и UUID пользователя не возвращаются.
- Публичное имя берётся из имени профиля и сокращается до имени с инициалом либо заменяется на «Покупатель Jardin Secret».
- Ограничения базы проверяют рейтинг, длину текста и уникальность `order_id`.

## Интерфейс

- На главной появляется секция «Отзывы покупателей» с последними опубликованными отзывами из Telegram и сайта.
- Страница `/reviews` объединяет оба источника, показывает среднюю оценку сайта, фильтр по оценке и ссылку на исходный Telegram-пост для импортированных отзывов.
- В `/account` появляется блок заказов. У выполненного заказа без отзыва показывается кнопка «Оставить отзыв».
- Страница `/account/orders/:orderId/review` содержит доступную клавиатурой оценку, поле текста и выбор аромата из заказа.
- На странице конкретного аромата показываются опубликованные отзывы, привязанные к этому товару.
- Пустые и ошибочные состояния объясняются коротким русским текстом; основной магазин остаётся доступным при временной недоступности отзывов.

## Проверка

- Parser-тесты подтверждают извлечение только текста, дедупликацию и отбрасывание медиа-публикаций без текста.
- Database-тесты подтверждают RLS, запрет отзыва до `completed`, один отзыв на заказ и публичность только `published`.
- Component-тесты проверяют список, рейтинг, состояния модерации и форму.
- E2E проверяет гостевой просмотр отзывов и доступ авторизованного пользователя к форме только у выполненного заказа.

## Публикация

- Telegram-отзывы импортируются в статический JSON и публикуются вместе с frontend.
- Пользовательские отзывы и заказы хранятся в Supabase.
- Vercel получает только публичные Supabase-переменные; служебный ключ не используется в браузере.
